## 🎓 今日深层认知

### 最深的理解：
Text-to-SQL 的本质是**信息传递**：LLM 需要通过 Schema 文档、Few-shot 示例、业务规则的清晰传达，才能理解"什么表"、"什么字段"、"什么规则"。
- Schema 文档 = "你有什么数据"
- Few-shot 示例 = "怎样用数据"
- 业务规则 = "为什么这样用"

这三者缺一不可。

### 挑战的认识：
- 准确率的提升是**多层次的**：不是单靠优秀的 Schema 就够，需要 Schema + Few-shot + 验证机制的组合
- 复杂任务可以**通过递进拆解**来解决（简→中→难）
- 成本和准确率需要**分层平衡**（不是所有查询都值得用 LLM）

### 与前面内容的联系：
- **Week 1（LLM 基础）**：理解为什么 LLM 会犯错（信息不足、上下文有限）
- **Week 2（Agent 框架）**：Few-shot 就像给 Agent 的"工具示例"，递进教学
- **Week 3（Prompt 工程）**：Schema 文档设计本质上就是高质量的 Prompt 工程
- **Week 4（RAG 系统）**：Schema 检索、Few-shot 示例的检索都涉及信息检索
