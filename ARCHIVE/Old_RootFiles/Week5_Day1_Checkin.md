### 最重要的3个概念

1. **任务拆分的本质**
   - 大问题分解 → 小问题 → 错误率↓，可追溯性↑
   - 这是 Agent 系统处理复杂问题的基础

2. **显式规划 vs 预定义工作流的权衡**
   - 显式规划：灵活但慢（每次都要 LLM 规划）
   - 预定义工作流：快但不灵活（固定流程）
   - 选择取决于"问题多样性"和"响应时间要求"

3. **企业级 Fallback 策略**
   - 不能依赖单一方案，需要分级降级
   - 从灵活 → 稳定 → 人工兜底
