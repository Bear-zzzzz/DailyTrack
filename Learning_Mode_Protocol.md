# 🎓 Agent 开发学习模式指南

> 建立于：2026-04-10
> 目的：确保学习质量、避免信息过载、逐步深化理解

---

## 📋 学习模式规则

### **原则**
1. ✅ **分模块教学**：一次只讲一个小主题（5-15 分钟读完）
2. ✅ **等待反馈**：必须等用户表示理解后才进行下一步
3. ✅ **提问验证**：通过 2-3 个检查题来确认掌握
4. ✅ **循序渐进**：只有完成某个部分，才能标记完成并进行下一部分
5. ✅ **质量优先**：宁可进度慢，也要确保理解到位

---

## 🔄 单个学习模块的完整流程

```
第 1 步：我给出模块概览
  ├─ 这个模块讲什么（1-2 句话）
  ├─ 为什么重要（1-2 句话）
  ├─ 预计阅读时间
  └─ 核心概念预告（3-5 点）

第 2 步：等待用户准备
  └─ 用户回复："准备好了" / "开始" / "可以了"

第 3 步：我给出核心学习资料
  ├─ 只给这个模块的内容
  ├─ 包括定义、公式、例子、代码（如有）
  └─ 清晰标注重点内容

第 4 步：等待用户阅读完成
  └─ 用户回复："看完了" / "已阅读" / "完成"

第 5 步：我进行检查提问
  ├─ 提出 2-3 个检查题（由易到难）
  ├─ 题目针对核心概念
  └─ 给出回答提示或完全开放

第 6 步：用户提交答案
  └─ 可以是完整回答，也可以是部分理解的提问

第 7 步：我进行评估和反馈
  ├─ 评估理解程度（0-100%）
  ├─ 如果理解 ≥ 80%：进行下一个部分
  ├─ 如果理解 < 80%：深化讲解或用另一个角度
  └─ 记录本模块的学习状态

第 8 步：完成该模块
  └─ 当理解到 80%+ 时，在进度文件中标记✅完成
```

---

## 📊 学习状态跟踪

每个模块有 4 种状态：

| 状态 | 符号 | 含义 | 操作 |
|------|------|------|------|
| 未开始 | ⬜ | 还没看资料 | 等待用户表示准备好 |
| 学习中 | 🟨 | 用户正在阅读 | 等待用户说"看完了" |
| 验证中 | 🟦 | 进行提问检查 | 等待用户回答 |
| 已完成 | ✅ | 理解 ≥ 80% | 进入下一个模块 |

## 🎯 我的承诺

在这个学习模式下，我会：

- [ ] **只给一个部分**，不会把整周的内容一次性倒出来
- [ ] **每次确认**你看完了再进行下一步
- [ ] **必须提问验证**，不会跳过检查环节
- [ ] **根据你的答案调整**教学深度和速度
- [ ] **记录每个模块的完成状态**，确保准确性
- [ ] **如果你不懂**，我会换个角度讲或深化讲解，直到你理解
- [ ] **只标记"完成"**当你真的掌握了（不是假完成）

---

## 📝 用户这边的承诺

希望你配合：

- [ ] **看完资料后及时反馈**（"看完了" 或 "阅读完成"）
- [ ] **认真回答检查题**（可以不完美，但要尽力）
- [ ] **有不懂的立即提出**（不要勉强进行）
- [ ] **遵循学习节奏**（不要跳过检查环节想加速）

## 🚦 学习速度预期

根据这个模式，学习节奏会是：

- **正常情况**：一个模块 15-20 分钟（包括提问和验证）
- **理解快的模块**：10-15 分钟
- **理解慢的模块**：20-30 分钟（可能需要深化讲解）
- **一周预计时间**：15-20 小时（保持原计划）
