# 第一部分：知识体系总结

## 1️⃣ Transformer 架构基础

### 核心架构
```
输入文本
  ↓
Embedding（词向量化）
  ↓
位置编码（加入位置信息）
  ↓
Transformer 层（多个）
  ├─ Self-Attention（学习词间关系）
  ├─ Feed-Forward Network（学习复杂特征）
  └─ Layer Norm + Residual Connection
  ↓
输出层
  ↓
生成下一个词
```

### 为什么比 RNN 好？
1. **并行化**：能并行处理整个序列（RNN 必须顺序）
2. **长距离依赖**：直接连接任意位置（RNN 会衰减）
3. **易于堆叠**：多层网络训练稳定

### Encoder vs Decoder
| 特性 | Encoder | Decoder |
|------|---------|---------|
| 视野 | 完全可见 | 掩码可见 |
| 任务 | 理解 | 生成 |
| 代表 | BERT | GPT |

## 2️⃣ Self-Attention（核心机制）

### 三步工作流
```
第 1 步：计算 Q、K、V
  Q = x · W_Q（查询）
  K = x · W_K（索引）
  V = x · W_V（内容）

第 2 步：计算注意力权重
  权重 = softmax(Q·K^T / √d_k)

第 3 步：加权求和
  输出 = 权重 · V
```

### 关键公式
```
Attention(Q, K, V) = softmax(Q · K^T / √d_k) · V
```

### 为什么除以 √d_k？
- 防止 Q·K^T 数值过大
- 避免 softmax 梯度消失
- 保证数值稳定性

---

## 3️⃣ Multi-Head Attention

### 设计思想
- 一个头学一种关系（语法、语义、指代等）
- 多个头学多种关系
- 最后融合所有头的信息

### 工作流
```
输入 → 分割成 8 份 → 并行计算 8 个 Attention
  ↓
拼接 → 线性投影(W_O) → 输出

特点：
- 8 个头是经验最优值
- 每头维度 = 总维度 / 头数（512/8=64）
- W_O 用来融合多头信息
```

---

## 4️⃣ Feed-Forward Network

### 结构
```
输入 (512维)
  ↓
升维 (512 → 2048)  # 线性变换 1
  ↓
ReLU激活  # 引入非线性
  ↓
降维 (2048 → 512)  # 线性变换 2
  ↓
输出
```

### 核心作用
- **Attention**：全局交互（开会议）
- **FFN**：局部深化（独立思考）
- 两者结合才能学到复杂的语言现象

### 为什么需要 ReLU？
- 线性层复合仍是线性
- ReLU 引入非线性
- 能学复杂函数

## 5️⃣ 位置编码

### 问题
- Attention 是对称的
- 无法感知词的顺序
- "狗咬人" vs "人咬狗" 无法区分

### 解决方案
```
在 embedding 中加入位置信息

公式：
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

特点：
- 低维高频：区分近距离
- 高维低频：区分远距离
- 支持任意长度序列
```

## 7️⃣ Function Call 原理

### 定义
```
Function Call = 模型调用外部工具的能力

工作流：
1. 定义工具（JSON Schema）
2. 模型理解用户请求
3. 模型决定调用哪个工具和参数
4. 系统执行工具
5. 模型基于结果组织答案
```

### 关键意义
- Chatbot（只生成文本）→ Agent（能执行操作）
- 从"回答问题"升级到"执行任务"

## 8️⃣ LoRA 微调

### 核心原理
```
原始：y = W·x
LoRA：y = (W + B·A)·x = W·x + ΔW·x

其中：
- W：预训练权重（冻结）
- ΔW = B·A：微调增量（学习）
- 秩 r 通常 8、16、32
```

### 参数对比
```
W：         100万参数
ΔW (r=8)：  16000参数（1.6%！）
```

### 为什么有效？
- 权重矩阵本身是低秩的
- 微调只需改变低维方向
- 高维方向改变贡献小

### LoRA vs Full Finetune
| 维度 | LoRA | Full Finetune |
|------|------|---|
| 参数 | 1.6% | 100% |
| 成本 | ¥ | ¥¥¥¥ |
| 速度 | 快 | 慢 |
| 效果 | 95%+ | 100% |

## 9️⃣ 分词（Tokenization）

### 为什么需要分词？
```
不分词的问题：
- 按字：词汇表太大
- 按词：无法处理新词

子词分割（现代方案）：
- "learning" → ["learn", "ing"]
- 能处理新词，词汇表适中
```

### 三种算法

#### BPE（Byte-Pair Encoding）
```
思想：从字符开始，统计合并最频繁的相邻对
特点：完全基于统计，对新词友好
使用：GPT 系列
```

#### WordPiece
```
思想：从词汇表开始，贪心分割未知词
特点：需要预定义词汇表
使用：BERT
```

#### SentencePiece
```
思想：端到端学习，不依赖预分词
特点：对任何语言一致
使用：T5、PaLM
```

### 实际影响
```
同一句话在不同模型中 token 数不同
原因：分割算法和词汇表不同
```

## 🔟 Embedding（词嵌入）

### 本质
```
Embedding = 查表过程

结构：
矩阵大小 = vocab_size × embedding_dim
每一行 = 一个 token 的向量
```

### embedding_dim 的意义
```
维度 = 能表达多少个"独立的语义维度"

维度大：
  ✓ 能编码复杂语义
  ✗ 参数多，计算量大

维度小：
  ✓ 参数少，快
  ✗ 信息编码能力弱

典型设置：
BERT/GPT-2：768
GPT-3：12,288
```

### 学习过程
```
Embedding 不是固定的，是通过梯度下降学出来的

预训练时：
- 随机初始化
- 经过数十亿次更新
- 学到语义关系
```
